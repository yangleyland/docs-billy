<Update label="May 2026" description="v4.2.0" tags={["Blackwell", "CuTe"]}>
  Added support for Blackwell's fifth-generation Tensor Core wgmma instructions with 2x register file throughput, unlocking FP64 matrix multiply at near-FP32 speeds.

  Introduced CuTe cluster-level collective operations for multi-SM tile reductions, enabling single-kernel AllReduce across up to 16 SMs without global memory round-trips.
</Update>

	birthday month!!

<Update label="March 2026" description="v4.1.0" tags={["Blackwell", "Python DSL"]}>
  Introduced native Blackwell FP4 tensor core support in the Python DSL, enabling sub-byte precision GEMM operations with up to 2x throughput improvements over FP8 on GB200 hardware.

  Added automatic kernel fusion for back-to-back GEMM operations targeting Blackwell's new asynchronous execution units.
</Update>

<Update label="February 2026" description="v4.0.2" tags={["Bug Fix", "CuTe"]}>
  Fixed a race condition in CuTe asynchronous copy operations when using TMA multicast with more than 4 consumer thread blocks.

  Resolved an issue where `tiled_mma` layout inference would produce incorrect results for non-power-of-two tile shapes on Hopper architectures.
</Update>

<Update label="January 2026" description="v4.0.0" tags={["Major Release", "Python DSL", "Hopper"]}>
  Major release introducing the CuTe DSL compiler backend, allowing Python-defined kernels to be ahead-of-time compiled into standalone CUDA libraries without a Python runtime dependency.

  Redesigned the epilogue API to support arbitrary DAG-structured epilogue fusions with automatic shared memory staging.

  Upgraded the default GEMM scheduling policy on Hopper to use persistent warp-specialized kernels, improving performance by 15-25% on large problem sizes.
</Update>

<Update label="December 2025" description="v3.8.0" tags={["Performance", "Ampere"]}>
  Backported warp-specialized GEMM scheduling to Ampere architectures for SM80 and SM86, yielding 10-18% speedups on A100 for narrow-tall matrix shapes.

  Added mixed-precision INT8xFP16 GEMM support with per-channel quantization scales in the epilogue.
</Update>

<Update label="November 2025" description="v3.7.1" tags={["Python DSL", "Bug Fix"]}>
  Fixed a memory leak in the Python DSL JIT cache that occurred when compiling more than 1024 unique kernel variants in a single process.

  Improved error messages for layout compatibility mismatches to include the actual and expected shapes.
</Update>

<Update label="October 2025" description="v3.7.0" tags={["Convolution", "Hopper"]}>
  Added implicit GEMM-based 3D convolution kernels with TMA-accelerated im2col for Hopper, supporting NDHWC and NCDHW data layouts.

  Introduced strided grouped convolution support with per-group bias and activation fusion.
</Update>

<Update label="September 2025" description="v3.6.0" tags={["CuTe", "Performance"]}>
  Overhauled the CuTe `make_tiled_copy` interface to automatically select optimal vectorization widths based on alignment and data type, reducing boilerplate by approximately 40%.

  Added L2 cache residency control hints for streaming GEMM workloads on Hopper, improving performance for bandwidth-bound problems by up to 20%.
</Update>
